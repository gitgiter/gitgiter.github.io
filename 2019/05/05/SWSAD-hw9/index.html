领域建模 - 对象状态

练习资源：Asg-RH.pdf

使用 UMLet 建模：

1、使用类图，分别对 Asg_RH 文档中 Make Reservation 用例以及 Payment 用例开展领域建模。然后，根据上述模型，给出建议的数据表以及主要字段，特别是主键和外键

  • 注意事项：
    • 对象必须是名词、特别是技术名词、报表、描述类的处理；
    • 关联必须有多重性、部分有名称与导航方向
    • 属性要注意计算字段

  • 数据建模，为了简化描述仅需要给出表清单，例如：

    • Hotel（ID/Key，Name，LoctionID/Fkey，Address…..）

  • Make Reservation

    • 领域模型
    • 数据库模型
      • Hotel (ID/Key, LocationID/FKey, Name, Stars, Address, Introduction, Rating, LowestPrice, IsFavourite, Discount)
      • Room (ID/Key, HotelID/FKey, Type, Date, Price, IsAvaliable)
      • RoomDescription(RoomDescriptionID/key, RoomCatalogID/Fkey, Type, Totals, List-price, description)
      • RoomCatalog(ID/key, Type)
      • Location (ID/Key, Name, IsFavourite, Code)
      • Traveler (ID/Key, Name, Phone, IDCardNum, Email)
      • Reservation (ID/Key, TravelerID/FKey, HotelID/FKey, NumOfRoom, TotalPrice)
      • ReservationItem (ID/Key, ReservationID/FKey, RoomID/Fkey, Type, CheckInDate, CheckOutDate, Price, NumOfNight, NumOfPerson)

  • Payment

    • 领域模型
    • 数据库模型
      • Payment (ID/Key, TimeStamp, TotalPrice)
      • PaymentItem (ID/Key, PaymentID/FKey, ReservationItemID/FKey, Price, Description, CreditCardType)
      • CreditCard (ID/Key, UserID/FKey, PaymentID/FKey, CardNumber, Type, SecurityCode, ExpiryDate)
      • CreditCardDetail (ID/Key, CreditCardID/FKey, FirstName, LastName, PostCode, CityOrState, Country)
      • User(ID/Key, Phone, IDCardNum, Email)

2、使用 UML State Model，对每个订单对象生命周期建模

  • 建模对象： 参考 Asg_RH 文档， 对 Reservation/Order 对象建模。
  • 建模要求： 参考练习不能提供足够信息帮助你对订单对象建模，请参考现在 定旅馆 的旅游网站，尽可能分析围绕订单发生的各种情况，直到订单通过销售事件（柜台销售）结束订单。
